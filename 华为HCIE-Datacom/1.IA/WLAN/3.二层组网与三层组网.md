```
什么是CAPWAP？  
CAPWAP（Control And Provisioning of Wireless Access Points Protocol Specification，无线接入点控制和配置协议）：  
该协议定义了如何对AP进行管理、业务配置，即AC通过CAPWAP隧道来实现对AP的集中管理和控制。
```
    
```
二层组网与三层组网的区别：  
AP-AC组网：  
二层是指AP和AC之间是二层组网，三层是指AC和AP之间是三层组网；  
二层组网AP可以通过二层广播，或者DHCP过程，即插即用上线；  
三层网络下，AP无法直接发现AC，需要通过DHCP或DNS方式动态发现，或者配置静态IP。  
在实际组网中：  
一台AC可以连接几十甚至几百台AP，组网一般比较复杂。  
比如在企业网络中，AP可以布放在办公室，会议室，会客间等场所，而AC可以安放在公司机房。  
这样，AP和AC之间的网络就是比较复杂的三层网络。  
因此，在大型组网中一般采用三层组网。
```

```
AC连接方式：  
直连模式下AC部署在用户的转发路径上，  
旁挂则相反；直连模式用户流量要经过AC，会消耗AC转发能力，旁挂一般流量不会经过AC。  
直连式组网：  
采用这种组网方式，对AC的吞吐量以及处理数据能力比较高，否则AC会是整个无线网络带宽的瓶颈。  
但用此种组网，组网架构清晰，组网实施起来简单。  
旁挂式组网：  
由于实际组网中，大部分不是早期就规划好无线网络，无线网络的覆盖架设大部分是后期在现有网络中扩展而来。  
而采用旁挂式组网就比较容易进行扩展，只需将AC旁挂在现有网络中，比如旁挂在汇聚交换机上，就可以对终端AP进行管理。  
所以此种组网方式使用率比较高。  
在旁挂式组网中，AC只承载对AP的管理功能，管理流封装在CAPWAP隧道中传输。数据业务流可以通过CAPWAP数据隧道经AC转发，也可以不经过AC转发直接转发，后者无线用户业务流经汇聚交换机由汇聚交换机传输至上层网络。
```

```
AP的转发模式：  
1.直连转发模式  
   AP接收终端数据后，直接按照业务VLAN的放行做数据转发  
   不会将数据转发到AC  
2.隧道转发模式  
   AP接收终端数据后，会将数据发送至AC，AC在执行数据转发￼￼直连转发和隧道转发的区别:￼1.相同点：￼    ac与ap之间的通信的管理vlan，连接链路上的所有交换机都要放行管理vlan￼2.不同点：￼    直连转发，转发的数据直接向外部转发，不需要回到ac，所以与外部连接的链路上需要放行对应的业务vlan￼    隧道转发，转发的数据需要先流向ac，再由ac转发到外部，即ac与ap连接的链路上的非与ac直连的交换机不许要配置多余的放行业务vlan；  
数据是如何转发的呢？  
sta通过ap封装业务vlan，再封装管理vlan，到ac，ac剥离管理vlan，再发往Internet，回收后再次封装业务vlan。￼   
```
 
```
无线二层组网：  
1.AC 和 AP之间连通  
1.AC的地址如何获取？  
管理员手工配置  
[AC6605]int vlanif 100   
   [AC6605-Vlanif100]ip address 172.16.100.254 24   
[AC6605]dhcp enable   
  [AC6605]int vlanif 100   
[AC6605-Vlanif100]dhcp select interface  
   2.AP的地址如何获取？  
自动获取的方式得到（AP不需要执行任何配置）AP的接口默认开启了自动获取地址的功能  
*AC和AP之间使用的VLAN，称为管理VLAN（AC管理AP的VLAN）  
    现场景中AC和AP的管理VLAN为100  
   *AC就相当于交换机设备  
   *AP就相当于终端设备（交换机连接AP的接口做trunk）  
      使用trunk完成ACCESS的功能  
    #  
    interface GigabitEthernet0/0/2  
     port link-type trunk  
     port trunk pvid vlan 100  
     port trunk allow-pass vlan 100  
    #
```
 
```
2.AC和AP之间建立capwap隧道  
   ***AC通过capwap隧道管理AP  
   1.AP向AC如何建立？  
      AP会默认发送广播的capwap报文，用于和同网段内所有AC建立capwap隧道  
   2.AC向AP如何建立？  
      1.AC需要确定向哪个AP建立capwap隧道  
        [AC6605]wlan  
        [AC6605-wlan-view]ap-id 1 ap-mac 00e0-fc2e-28b0  
        [AC6605-wlan-ap-1]ap-name AP111  改变AP的名称  
      2.AC需要指定发送capwap报文的源地址  
        [AC6605]capwap source interface Vlanif 100   
------------------------AP上线--------------------------  
  在AC设备上通过 display ap all 查看AP的状态为nor
```

```
3.AC给AP的业务配置：  
   1.信号模板分配  
     [AC6605]wlan

     [AC6605-wlan-ssid-prof-1]ssid test

  [AC6605]wlan  
  [AC6605-wlan-view]security-profile name 1  
  [AC6605-wlan-sec-prof-1]security wpa psk pass-phrase Huawei@123 aes  
  3.VAP模板绑定  
   [AC6605]wlan  
   [AC6605-wlan-view]vap-profile name 1  
   [AC6605-wlan-vap-prof-1]ssid-profile 1  
   [AC6605-wlan-vap-prof-1]security-profile 1   
   [AC6605-wlan-vap-prof-1]service-vlan vlan-id 10  
# 设置vlan业务id                                    
 [AC6605-wlan-vap-prof-1]forward-mode direct-forward    
# 默认转发方式：直接转发  
 4.在AP组中绑定 AP 和 VAP模板  
 [AC6605]wlan  
 [AC6605-wlan-view]ap-group name 1    创建AP组  
 [AC6605-wlan-ap-group-1]vap-profile 1 wlan 1 radio 0     
# 关联VAP模板并分配信号  
  [AC6605-wlan-view]ap-id 1      进入AP试图  
  [AC6605-wlan-ap-1]ap-group 1   将AP加入到AP组中
```
