动态路由 相比于 静态路由的优势：  
1.对于路由的计算能力 快  
2.对于网络故障的恢复能力 强  
3.对于管理员的运维 方便
```

```
OSPF的简单要求以及部分特性：  
OSPF的RID，要求两端设备不能相同  
OSPF的Area id，要求两端设备一致  
OSPF的报文：hello、DD、LSR、LSU、LSAck  
OSPF的状态机：down、init、two-way、exstart、exchange、loadding、full  
OSPF的邻居：通过hello报文交互形成邻居关系  
OSPF的DR、BDR：减少广播域内LSA的泛洪   
（接口优先级、RID）  
（DR失效、BDR一定成为DR）  
OSPF的主从关系：有序的报文交互机制 （RID大的）  
OSPF的邻接：通过LSU报文同步所有LSA，完善LSDB后成为邻接关系
```
 
```
OSPF区域内的路由计算过程：  
OSPF设备通过接收邻居发送的LSA信息，计算出拓扑信息和路由条目
```

```
LSA的信息内容：  
 1.LSA的头部:  每一类LSA的头部字段信息都相同  
Type（链路状态类型）： 指示本LSA的类型。  
Ls id（链路状态ID）：不同的LSA，对该字段的定义不同。   
Adv rtr（通告路由器）：产生该LSA的路由器的Router ID  
Ls age （链路状态老化时间）：表示LSA已经生存的时间，单位是秒。  
Len：是一个包含LSA头部在内的LSA的总长度值。  
Options（可选项）：每一个bit都对应了OSPF所支持的某种特性。  
seq#（链路状态序列号）：当LSA每次有新的实例产生时，序列号就会增加。  
chksum（校验和）：用于保证数据的完整性和准确性。
