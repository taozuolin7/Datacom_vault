```
在ISIS视图下 可以使用filter-policy进行过滤  
  filter-policy import：对于接收的LSP，计算路由时是否加入路由表进行过滤  
   [AR3]acl 2000   
   [AR3-acl-basic-2000]rule 5 deny source 1.1.1.1 0   
   [AR3-acl-basic-2000]rule 10 permit   
   [AR3]isis   
   [AR3-isis-1]filter-policy 2000 import  
  如果在L1/2设备上执行该操作，则对于过滤的路由信息不会在通过渗透产生对应的LSP携带了  
*所有设备都能使用￼  
  filter-policy export：对于外部路由在发出是进行过滤  
   [AR4]acl 2000  
   [AR4-acl-basic-2000]rule 5 deny source 100.4.4.4 0   
   [AR4-acl-basic-2000]rule 10 permit   
   [AR4]isis   
   [AR4-isis-1]filter-policy 2000 export   
   *该操作只能配置在引入外部路由的设备上，在其他的设备上执行该配置 没有效果  
   引入外部路由的设备，配置操作完成后，通告的分片LSP中不携带被过滤的路由信息
```
 
```
  filter-policy 过滤：L1/2设备在路由渗透时进行的过滤操作  
   [AR2]acl 2001  
   [AR2-acl-basic-2001]rule 5 deny source 3.3.3.3 0  
   [AR2-acl-basic-2001]rule 10 deny source 192.168.4.4 0   
   [AR2-acl-basic-2001]rule 15 permit   
   [AR2]isis   
   [AR2-isis-1]import-route isis level-2 into level-1 filter-policy 2001   
   对于渗透时的过滤，可以在L2渗透到L1时过滤，也可以在L1渗透到L2时过滤
```

  *ISIS是不区分内外部路由的
