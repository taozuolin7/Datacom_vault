```
ISIS认证：  
1.接口认证：对于ISIS的hello报文认证（在邻居建立时认证）  
  [AR1-GigabitEthernet0/0/0]isis authentication-mode md5 plain Huawei@123 
```
 
```
  hello报文中携带的认证字段：  
  Authentication (t=10, l=17)  
    Type: 10  
    Length: 17  
    hmac-md5 (54), message digest (length 16) = 26b57cb063ff81d361bb321b4df09c91
```
 
```
  可以在接口认证时选择参数指定是L1的hello报文 还是L2的hello报文携带认证
```
 
```
  也可以为认证设置send-only参数，该参数表示： （在密码替换时可以使用该参数实现邻居无缝切换）  
          1.本端发送hello报文携带认证参数  
          2.本端接收hello报文不检查认证参数
```
 
```
2.区域认证：对ISIS-L1的LSP和SNP报文的认证 （在LSDB同步时认证）  
  [AR1-isis-1]area-authentication-mode md5 plain Huawei@123 
```
 
```
  [AR1-isis-1]area-authentication-mode md5 plain Huawei@123   
    all-send-only   发送LSP和SNP报文时携带认证；接收LSP和SNP报文时不检查认证  
    snp-packet      LSP报文做认证，对SNP报文选择是否做认证
```
 
```
  [AR1-isis-1]area-authentication-mode md5 plain Huawei@123 snp-packet   
    authentication-avoid  发送时不携带认证，接收时不做认证检查  
    send-only             发送时携带认证，接收时不检查认证  
    
  做区域认证时，默认对LSP和SNP报文进行认证  
  LSP/SNP报文中的认证：  
  Authentication (t=10, l=17)  
    Type: 10  
    Length: 17  
    hmac-md5 (54), message digest (length 16) = 79d71ca4a3fc7d087d393c1d9c4a877f
```
   

```
3.路由域认证：对ISIS-L2的LSP和CSP报文的认证  
   和区域认证配置相同，选择的参数也相同，只是针对的报文级别发生了变化  
   [AR4-isis-1]domain-authentication-mode md5 plain Huawei@123
```
