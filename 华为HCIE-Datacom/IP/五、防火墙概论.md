```
防火墙安全区域的类型：  
1.系统安全区域  
   1.local：优先级值为100，相当于防火墙的自身，防火墙所有的接口都属于local区域  
   2.trust：优先级值为85，一般用于链接站点内  
   3.untrust：优先级值为5，一般用于链接外部  
   4.dmz：优先级值为50，一般用于链接服务区域  
2.自定义安全区域  
   #  
   firewall zone name test id 4  
    set priority 70  
   每一个安全区域都要存在优先级值，安全区域之间的优先级值不能冲突
```
 
```
***安全区域之间没有本质的区别，都是为了隔离  
  （优先级值没有可比性，不存在越大优先，或越小越优先）
```
 
```
*安全区域之间的流量默认是隔离的  
*安全区域内的流量默认是放行的
```
 
```
安全区域之间的流量访问：  
 需要通过安全策略的放行￼
```

```
什么是安全策略？  
  安全访问的部署策略（防火墙默认存在的访问策略）  
安全策略默认是拒绝所有的  
[USG6000V1]security-policy  进入安全策略试图  
[USG6000V1-policy-security]default action deny 	    
[USG6000V1-policy-security]default action permit   
安全策略可以定义明细的访问条目  
[USG6000V1-policy-security]rule name PC1-PC3  
条目内要编辑明细的流量信息  
[USG6000V1-policy-security-rule-PC1-PC3]source-zone trust  
[USG6000V1-policy-security-rule-PC1-PC3]destination-zone untrust   
[USG6000V1-policy-security-rule-PC1-PC3]source-address 192.168.1.1 32  
[USG6000V1-policy-security-rule-PC1-PC3]destination-address 192.168.3.3 32   
[USG6000V1-policy-security-rule-PC1-PC3]service icmp   
为条目添加执行的动作  
[USG6000V1-policy-security-rule-PC1-PC3]action permit 
```
 
```
该策略部署完毕后，PC1可以正常访问PC3，而PC3不能访问PC1
```
