2G 软交换网是将**3G R4 版本的软交换技术**应用于**2G (GSM) 核心网电路域 (CS)**的网络架构，核心是实现**控制与承载分离**，将传统 MSC (移动交换中心) 拆分为**MSC Server (控制面)**和**MGW (媒体网关，承载面)**，同时可引入 IP 承载网提升网络灵活性与效率。

---

### 一、核心定义与主要功能

**2G 软交换网定义**：在 2G GSM 网络基础上，引入移动软交换技术，将核心网电路域的呼叫控制功能与媒体交换功能分离，形成以 MSC Server 为控制核心、MGW 为媒体处理核心，可支持 TDM/ATM/IP 多种承载方式的 2G 核心网架构。

**主要用途**：

1. **基础语音业务**：提供 2G 用户之间、2G 与固网 / PSTN 用户之间的语音呼叫连接与控制
2. **补充业务**：支持呼叫转移、呼叫等待、三方通话、来电显示等传统 GSM 补充业务
3. **短信业务**：处理 2G 用户的短信收发控制与路由
4. **移动性管理**：负责用户位置更新、切换控制、鉴权认证等核心移动网络功能
5. **网络互通**：实现与传统 TDM 交换网、3G/4G 核心网及其他运营商网络的互联互通

---

### 二、核心作用与价值

|核心作用|具体价值|
|---|---|
|**控制与承载分离**|提高网络资源利用率，控制面与承载面独立扩容，降低建设与运维成本|
|**业务与网络解耦**|业务升级只需更新 MSC Server，无需改动 MGW，加速新业务部署|
|**网络融合基础**|为 2G/3G 共核心网、向 IMS 演进提供平滑过渡路径|
|**集中化管理**|可实现省级集中控制，整合维护资源，降低运维复杂度|
|**承载多样化**|支持从 TDM 向 IP 承载迁移，提升带宽利用率，降低传输成本|
|**资源共享**|MGW 可共享媒体处理资源 (如 TC、EC、放音收号)，提高设备利用率|

---

### 三、核心特点

1. **分层架构**：采用**业务层、控制层、承载层、接入层**的开放架构，各层独立演进
2. **功能模块化**：
    
    - MSC Server：专注呼叫控制、移动性管理、信令处理，无媒体交换功能
    - MGW：专注媒体流交换、编解码转换、回声抑制、资源提供
    - SG (信令网关)：负责 SS7 信令与 SIGTRAN 协议转换，适配 IP 承载网
    
3. **协议标准化**：采用国际标准协议，如 H.248/MEGACO (MSC Server 与 MGW 间)、SIGTRAN (信令传输)、MAP (移动应用部分) 等，提升设备互通性
4. **平滑演进**：可重用原 2G 网络 HLR、SCP、SMS 等网元，无线侧仍为 2G BSS 接入，与传统 TDM 交换网无缝互通
5. **灵活扩容**：MGW 扩容只需硬件扩容和增开电路，无需复杂数据配置，缩短工程周期

---

### 四、关键技术与网元

#### 1. 核心网元

|网元名称|英文全称|核心功能|
|---|---|---|
|MSC Server|Mobile Switching Center Server|呼叫控制、移动性管理、MGW 控制、业务逻辑执行|
|MGW|Media Gateway|媒体流交换、编解码转换 (TC)、回声抑制 (EC)、放音收号、A 接口接入|
|SG|Signaling Gateway|SS7 与 SIGTRAN 协议转换，信令在 IP 网上传输|
|HLR|Home Location Register|用户签约数据存储、鉴权中心 (AUC) 功能|
|SCP|Service Control Point|智能网业务控制，如预付费、彩铃等|
|SMSC|Short Message Service Center|短信存储转发中心|

#### 2. 核心技术

1. **控制与承载分离技术**：3GPP R4 规范的核心，是软交换技术的基础
2. **媒体网关控制技术**：通过 H.248/MEGACO 协议实现 MSC Server 对 MGW 的控制
3. **信令传输技术**：
    
    - SIGTRAN 协议族：将 SS7 信令适配到 IP 网络，包括 M3UA (应用层适配)、SCTP (流控制传输协议)、IP (网络层)
    - SS7 协议：用于与传统 TDM 网络互通及移动网内部信令
    
4. **IP 承载技术**：支持 IP 网络传输媒体流和信令，提高带宽利用率，降低成本
5. **移动性管理技术**：继承 GSM 核心移动性管理功能，如位置更新、切换控制、鉴权认证等
6. **业务提供技术**：通过 MSC Server 与 SSP 统一设置，实现业务集中处理与快速部署

---

### 五、与传统 2G TDM 交换网的核心区别

|对比项|2G 软交换网|传统 2G TDM 交换网|
|---|---|---|
|核心架构|控制与承载分离，MSC Server+MGW|控制与承载合一，单一 MSC 设备|
|承载方式|支持 TDM/ATM/IP 多承载|仅支持 TDM 承载|
|扩容方式|控制面与承载面独立扩容|必须整体扩容|
|业务部署|业务与网络解耦，升级快|业务与网络绑定，升级复杂|
|资源利用|媒体资源共享，利用率高|设备资源独立，利用率低|
|演进能力|支持向 3G/4G/IMS 平滑演进|演进能力有限，难以适应新业务|

---

### 总结

2G 软交换网是 2G 核心网的一次重要技术升级，核心是通过**控制与承载分离**实现网络架构优化，带来资源利用率提升、运维成本降低、业务部署加速等多重价值，同时为移动通信网络向 3G、4G 乃至 IMS 的演进奠定了坚实基础，是 2G 网络向 NGN (下一代网络) 过渡的关键技术路径。
