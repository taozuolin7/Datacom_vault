#### 第一章 绪论（约 3000 字）

本章核心是告诉评委 “为什么要做这个课题”，立住研究的意义和价值，避免空泛。

1.1 研究背景与研究意义

- 研究背景：智慧园区 / 数字化校园的发展趋势，园区网终端规模爆发、业务灵活化的需求；传统园区网基于 VLAN 的技术瓶颈（4096 数量限制、广播域过大、跨三层终端迁移业务中断、业务隔离不灵活、运维复杂度高）；VXLAN 技术作为大二层网络核心技术的优势，以及集中式网关部署模式在中小园区的适配性。
- 研究意义：理论意义（完善 VXLAN 技术在园区网场景的落地应用方案，补充集中式网关部署的实践参考）；实践意义（解决传统园区网的核心痛点，设计一套易运维、高可靠、可扩展的园区网方案，为同类型高校 / 企业园区网建设提供参考）；个人意义（掌握数据中心级主流网络技术，提升专业能力，匹配行业岗位需求）。
    
    1.2 国内外研究现状
- 国外研究：VXLAN 技术的标准化（RFC7348），Cisco、Juniper 等厂商从数据中心向园区网延伸的 VXLAN 解决方案，相关学术研究进展。
- 国内研究：华为、华三为代表的厂商推出的园区 VXLAN 商用方案，国内高校、企业对 VXLAN 园区网的应用与研究，集中式网关在中小园区的落地优势分析。
- 现状总结：当前 VXLAN 研究多集中于数据中心分布式网关场景，针对中小园区的集中式网关 VXLAN 方案的精细化设计与落地验证不足，这也是你课题的研究切入点。
    
    1.3 主要研究内容与论文结构
- 主要研究内容：①园区网需求分析与设计原则制定；②集中式网关 VXLAN 园区网整体方案设计；③方案的仿真配置与实现；④方案的功能性、可靠性、性能测试与结果分析。
- 论文结构：逐章简要说明核心内容，让评委快速了解全文逻辑。
    
    1.4 本章小结

#### 第二章 相关技术理论基础（约 4000 字）

本章是论文的理论支撑，也是答辩评委的重点提问区，**严禁直接抄厂商文档 / 百科，要用自己的话结合你的课题场景复述**，重点突出和你的方案强相关的技术。

2.1 VXLAN 核心技术原理

- VXLAN 基本概念：MAC-in-UDP 封装格式、24 位 VNI（VXLAN 网络标识符，解决 VLAN 数量限制）、VTEP（VXLAN 隧道端点，核心封装 / 解封装设备）、BD（桥接域，对应二层广播域）、VXLAN 隧道的核心定义。
- VXLAN 核心转发机制：同子网二层转发流程、BUM 流量（广播、未知单播、组播）的转发机制。
    
    2.2 VXLAN 网关部署模式（本章核心，必须突出你的课题重点）
- 集中式 VXLAN 网关：核心定义（所有租户的三层网关集中部署在核心 / Spine 节点，Leaf 节点仅做二层 VTEP 转发）、转发流程、核心优缺点、适用场景（中小园区、集中管控需求、简化运维）。
- 分布式 VXLAN 网关：核心定义、转发流程、优缺点、适用场景，**重点做两种模式的对比**，明确你的课题选择集中式网关的原因（贴合园区规模、运维成本低、策略集中管控、符合中小园区建设需求）。
    
    2.3 BGP EVPN 控制平面技术
- 为什么用 BGP EVPN：对比静态 VXLAN 的缺陷（手工配置扩展性差、广播泛洪严重、无法实现终端平滑迁移），BGP EVPN 作为 VXLAN 控制平面的核心优势（VTEP 自动发现、MAC/IP 路由自动同步、ARP 代答减少泛洪、支持终端无缝迁移）。
- BGP EVPN 核心路由类型：重点讲和你的方案相关的 3 类（2 类 MAC/IP 路由、3 类集成多播路由、5 类 IP 前缀路由），结合你的园区网场景说明其作用。
    
    2.4 园区网配套支撑技术
- 简要介绍你方案中用到的技术：底层 IGP（OSPF）、链路聚合 Eth-Trunk、可靠性技术（VRRP/BFD）、接入安全技术（802.1X / 端口安全）、无线有线一体化技术（CAPWAP）等，只讲原理，不展开配置。
    
    2.5 本章小结

#### 第三章 园区网需求分析（约 2500 字）

本章是你所有设计的依据，**严禁凭空设计，所有后续设计都必须紧扣本章的需求**，避免设计和需求脱节。

3.1 项目概述

- 明确设计对象：推荐选择**高校新校区园区网**（贴合你的学生身份，场景熟悉，需求明确），说明园区覆盖范围（教学楼、实验楼、办公楼、学生宿舍、图书馆、数据中心、出口机房）、终端规模（有线终端 5000+、无线终端 10000+）、业务类型（教学业务、办公业务、学生宿舍业务、访客业务、数据中心业务）。
    
    3.2 功能性需求（核心，逐条对应后续设计）
- 核心需求 1：大二层隔离与扩展需求：突破 VLAN 数量限制，实现不同业务 / 租户的二层隔离，满足多业务灵活划分需求。
- 核心需求 2：集中式网关与统一管控需求：三层网关集中部署，实现跨子网流量的集中转发、策略统一配置，简化园区运维难度。
- 核心需求 3：终端无缝迁移需求：终端在园区内跨区域移动，IP 地址保持不变，业务不中断，满足移动办公、无线漫游的需求。
- 核心需求 4：高可靠性需求：网络无单点故障，链路 / 设备故障时实现快速切换，保障业务连续。
- 核心需求 5：安全管控需求：实现不同业务租户的隔离与精细化访问控制，满足非法接入防护、边界安全防护需求。
- 扩展需求：无线有线一体化管控、QoS 服务质量保障、统一运维管理需求。
    
    3.3 非功能性需求
- 性能需求：核心链路带宽 10G，接入链路带宽 1G，端到端转发时延≤50ms，正常场景丢包率为 0。
- 可扩展性需求：网络架构支持未来终端规模、业务类型的扩展，配置可平滑调整。
- 可运维性需求：配置简化，故障排查便捷，支持统一管理。
- 安全性需求：符合网络安全等级保护 2.0 二级相关要求。
    
    3.4 方案设计原则
- 明确设计的核心原则：实用性、可靠性、安全性、可扩展性、易运维性、经济性，所有设计都遵循该原则。
    
    3.5 本章小结

#### 第四章 集中式网关 VXLAN 园区网整体方案设计（约 5000 字，论文核心章节，占比最大）

本章是毕设的核心，要做到**设计有依据、细节可落地、逻辑全闭环**，所有设计都要对应第三章的需求，重点突出 “集中式网关” 的设计核心。

4.1 网络整体架构设计

- 整体架构选型：采用**Spine-Leaf 二层 CLOS 架构**（贴合 VXLAN 技术架构，行业主流），明确各层级的角色与职责：
    
    - Spine 层（核心层）：2 台冗余部署，作为全网核心转发节点、BGP EVPN 路由反射器 RR、**集中式三层网关**，是整个 VXLAN 网络的核心。
    - Leaf 层（接入层）：按区域划分为接入 Leaf（教学楼、宿舍、办公等区域，双机冗余）、服务 Leaf（连接数据中心、出口防火墙、AC 控制器、互联网出口），所有 Leaf 节点作为 VTEP，实现 VXLAN 二层封装 / 解封装，终端接入。
    
- 物理拓扑设计：附完整的拓扑图，标注设备型号、角色、互联链路类型，明确冗余设计。
    
    4.2 地址与标识规划设计
- IP 地址规划：Loopback0 地址规划（用于 BGP EVPN 对等体、VXLAN 隧道源地址）、设备管理地址规划、互联链路地址规划、业务网段地址规划（每个 BD 对应一个业务子网），附完整的地址规划表。
- VNI 与 VLAN 规划：每个业务 / 租户分配唯一 VNI，明确 VNI-BD-VLAN 的一一映射关系，附规划表，明确规划原则。
    
    4.3 VXLAN 控制平面与数据平面设计
- 控制平面设计：BGP EVPN 部署方案，Spine 节点作为路由反射器 RR，所有 Leaf 节点与 Spine 建立 IBGP EVPN 对等体；BUM 流量采用头端复制方案（中小园区无需组播，配置简化，易落地）。
- 数据平面设计：明确 VXLAN 封装规则，**重点绘制并讲解集中式网关的核心转发流程**：①同子网跨 Leaf 终端二层转发流程；②跨子网终端集中式网关三层转发流程，这是你课题的核心，必须讲透。
    
    4.4 集中式三层网关专项设计（本章重中之重，突出题目核心）
- 网关冗余设计：双 Spine 节点部署双活集中式网关，采用 VRRP+VBDIF 接口设计，虚拟网关 IP 统一，双节点均可实现三层转发，避免单点故障。
- 网关功能设计：集中式网关上实现 VBDIF 接口配置（对应每个租户的三层网关）、ARP 代答、DHCP 中继、精细化 ACL 访问控制、QoS 标记等核心功能。
- 路由设计：明确租户 VPN 实例规划，集中式网关的路由发布与转发规则，园区内路由与出口路由的对接方案。
    
    4.5 可靠性方案设计
- 设备级冗余：Spine、Leaf 节点均双机冗余部署，无单点故障。
- 链路级冗余：所有互联链路均采用 Eth-Trunk 链路聚合，提升带宽与冗余能力。
- 协议级可靠性：BFD 联动 OSPF/BGP/VRRP，故障收敛时间控制在 500ms 以内；VRRP 双机热备，实现网关冗余切换。
    
    4.6 安全与配套方案设计
- 安全方案设计：租户二层隔离（不同 VNI 默认不可互访）、集中式网关三层 ACL 精细化访问控制、接入安全（802.1X / 端口安全）、边界安全（防火墙 / NAT / 入侵防御）、管理安全（带外管理 / SSH 权限管控）。
- 无线有线一体化设计：AC 控制器接入服务 Leaf，AP 通过 CAPWAP 隧道接入 AC，无线用户流量统一纳入 VXLAN 隧道，实现无线有线用户的统一管控、统一策略。
- QoS 方案设计：针对教学、语音、视频、办公等业务划分优先级，实现拥塞管理与流量整形。
    
    4.7 本章小结

## 三、核心避坑指南与亮点打造

### 必避的 5 个核心坑

1. **核心概念错误**：集中式网关的核心是 “三层网关全部集中在 Spine / 核心节点，Leaf 仅做二层 VTEP”，不要在 Leaf 上配置三层网关，否则直接偏离题目核心，答辩必翻车。
2. **只谈设计不做实现**：本科毕设非常看重 “动手能力”，只写理论设计没有仿真、没有配置、没有测试，大概率被评委质疑工作量不足，分数偏低。
3. **场景跑偏**：不要把园区网 VXLAN 做成数据中心 VXLAN，要突出园区网的特点：终端移动性、无线有线一体化、接入安全、多租户业务隔离，否则会被评委认为对场景理解不足。
4. **控制平面选静态 VXLAN**：静态 VXLAN 配置简单，但没有技术亮点，工作量不足，BGP EVPN 是当前行业主流，也是毕设的核心加分项，必须选用。
5. **论文逻辑脱节**：需求 - 设计 - 实现 - 测试必须一一对应，设计的功能必须有需求支撑，测试的内容必须对应设计的功能，避免前后脱节，东拼西凑。

### 3 个低成本高收益的亮点打造

1. **抓包分析 VXLAN 封装**：在仿真环境中用 Wireshark 抓取 VXLAN 封装的报文，在论文中展示并分析封装结构，证明你真的理解了 VXLAN 的转发原理，答辩时非常加分。
2. **对比传统园区网与 VXLAN 园区网的性能**：在仿真环境中搭建传统 VLAN 三层园区网，和你的 VXLAN 园区网做转发时延、带宽、终端迁移收敛时间的对比测试，用数据证明 VXLAN 方案的优势，提升论文的深度。
3. **贴合高校场景做精细化设计**：针对高校的教学区、宿舍区、办公区、访客区做差异化的策略设计，比如教学区带宽保障、宿舍区流量限速、访客区隔离，让方案更贴合实际，不是空泛的技术堆砌。
