
# 中国移动网络层级与设备角色全解析（从局域网到互联网）

要理解这些设备的关系，核心是先理清 **“用户侧→接入网→城域网→省骨干网→全国骨干网→网间互联”** 的网络层级逻辑，设备的角色完全对应其所在层级的功能（如接入、汇聚、核心、互联）。以下结合中国移动实际网络架构，用 “层级 + 设备 + 功能” 的结构化方式拆解，再讲数据转发路径：

## 一、先明确核心网络层级（从近到远，从用户到互联网）

|网络层级|核心作用|覆盖范围|关键设备（后文详解）|
|---|---|---|---|
|局域网（LAN）|用户 / 企业内部组网（如家庭、公司、园区）|单栋楼、园区、企业内部|交换机（二层）、AR（接入路由器）|
|接入网|将局域网接入运营商网络（“最后一公里”）|小区、写字楼、乡镇|BRAS、ONU/OLT（光纤接入）、AR|
|城域网|汇聚本市接入网流量，转发到省骨干网|地级市 / 县级市|PA（城域汇聚）、MB（移动回传 / 云接入）、BR（城域边界）|
|省骨干网|汇聚全省城域网流量，连接全国骨干网|省级范围|PB（省骨干汇聚）、PC（省骨干核心）、CR（省核心路由器）|
|全国骨干网|连接各省骨干网，承载全国跨域流量|全国范围|NCR（全国核心路由器）、骨干 CR|
|国内网间互联|与中国电信、联通等其他运营商互通|国家级互联节点（如北京、上海）|网间 BR（边界路由器）、IXP（互联网交换中心）|
|国外网间互联|与国际运营商 / 国际互联网互通|国际出口节点（如广州、香港）|国际出口路由器、NCR|

注：你提到的 **CMnet（公众互联网）、IP 承载网、移动云专网** 是 “业务承载网络”，而非独立层级 —— 它们共用上述物理层级，只是转发路径和优先级不同（如 CMnet 走公众路由，云专网走专属路由）。

## 二、关键设备角色详解（按层级对应）

### 1. 局域网（用户侧）核心设备222

- **AR（Access Router，接入路由器）**
    
    - 角色：企业 / 园区局域网的 “出口网关”，也是小型局域网接入运营商网络的核心设备。
    - 功能：① 实现局域网内部 IP 分配（DHCP）、路由转发；② 连接运营商网络（如通过专线或宽带连接到 BRAS）；③ 简单安全控制（防火墙、ACL）。
    - 举例：公司园区的出口路由器，家庭宽带若用企业专线，也可能通过 AR 接入。
- 补充：局域网内还有 **二层交换机**（负责终端互联，如电脑、服务器），但不属于运营商设备，是用户自有设备。
    

### 2. 接入网核心设备

- **BRAS（Broadband Remote Access Server，宽带远程接入服务器）**
    
    - 角色：运营商接入网的 “入口大门”，所有宽带用户（家庭、普通企业）的流量都需经过 BRAS 接入。
    - 核心功能：① 用户认证 / 授权（如输入移动宽带账号密码，由 BRAS 验证）；② 带宽控制（如 100M/1000M 宽带限速）；③ 流量汇聚（将多个小区 / 写字楼的用户流量汇总到城域网）；④ 地址转换（NAT，将用户私网 IP 转为公网 IP）。
    - 位置：通常部署在城市的 “接入机房”，靠近用户侧。
- 补充：光纤接入场景中，还有 **ONU（用户侧光猫）、OLT（局端光线路终端）**，负责光纤信号的光电转换，最终流量会汇总到 BRAS。
    

### 3. 城域网核心设备

- **PA（Aggregation Router，城域汇聚路由器）**
    
    - 角色：城域网的 “流量中转站”，汇聚多个 BRAS 或接入节点的流量。
    - 功能：① 收敛接入网的分散流量，减少核心层设备的压力；② 简单路由策略（如过滤无效流量）；③ 连接城域网核心设备（MB/BR）。
- **MB（Mobile Backhaul/Mobile Cloud Router，移动回传 / 移动云接入路由器）**
    
    - 角色：城域网的 “多功能节点”，分两种场景：
        
        ① 移动回传：承载 5G/4G 基站的流量回传至核心网；
        
        ② 移动云专网：专门对接移动云资源，为企业用户提供 “局域网→移动云” 的专属通道（对应你提到的 “移动云专网” 访问方式）。
    - 位置：部署在城市核心机房，连接 PA 和省骨干网入口。
- **BR（Border Router，城域边界路由器）**
    
    - 角色：城域网的 “出口路由器”，负责将城域网流量转发到省骨干网。
    - 功能：① 城域网与省骨干网的路由互联；② 流量隔离（如区分 CMnet 和 IP 承载网流量）；③ 城域级别的安全防护（如抵御 DDoS 攻击）。

### 4. 省骨干网核心设备

- **PB（Province Backbone Aggregation，省骨干汇聚路由器）**
    
    - 角色：省骨干网的 “区域汇聚点”，汇聚全省各地市城域网的流量（通过 BR 接入）。
    - 功能：① 收敛全省城域流量，避免省核心设备直接对接大量城域节点；② 省域内路由汇总（如将某市的明细路由汇总为网段路由）；③ 连接省核心设备（PC/CR）。
- **PC（Province Core Router，省骨干核心路由器）**
    
    - 角色：省骨干网的 “核心交换节点”，相当于 “省级交通枢纽”。
    - 功能：① 省域内跨地市流量转发（如成都用户访问绵阳服务器）；② 连接全国骨干网（通过 NCR）；③ 承载全省重要业务（如政务专线、企业 VPN）。
- **CR（Core Router，骨干核心路由器）**
    
    - 角色：省骨干网与全国骨干网的 “衔接节点”，部分场景下与 PC 功能重合（中国移动部分省份将 PC 称为 CR）。
    - 核心差异：CR 更侧重 “跨省互联”，直接上联全国骨干网的 NCR；PC 侧重 “省内核心转发”。

### 5. 全国骨干网与网间互联设备

- **NCR（National Core Router，全国核心路由器）**
    
    - 角色：中国移动全国网络的 “顶级枢纽”，相当于 “国家交通总枢纽”。
    - 功能：① 连接各省骨干网的 CR/PC（实现跨省流量转发，如四川用户访问广东服务器）；② 连接国内其他运营商（电信、联通）的互联节点；③ 连接国际出口路由器（实现跨国访问）；④ 承载全国性业务（如 CMnet 骨干、IP 承载网核心）。
- **网间 BR（Border Router，网间边界路由器）**
    
    - 角色：运营商之间的 “流量关口”，部署在国内网间互联节点（如北京、上海的 IXP 互联网交换中心）。
    - 功能：① 与电信、联通等运营商的 BR 对等互联，交换网间流量（如移动用户访问电信服务器）；② 网间路由控制和带宽管理；③ 网间安全防护（如过滤跨运营商攻击流量）。
- 国际出口路由器：部署在国际出口节点（如广州、香港），连接国际运营商（如 NTT、Level3），实现跨国访问（如移动用户访问谷歌、亚马逊）。
    

## 三、关键设备易混淆点区分（避免踩坑）

|设备对|核心区别|
|---|---|
|BRAS vs AR|BRAS 是运营商侧 “公众宽带接入网关”（面向海量家庭 / 普通用户）；AR 是用户侧 “企业 / 园区出口网关”（面向企业专线用户）|
|BR vs BRAS|BR 是 “边界路由器”（城域 / 网间的路由互联）；BRAS 是 “接入服务器”（用户认证 + 带宽控制）|
|CR vs NCR|CR 是 “省骨干核心”（省内 / 省际接入）；NCR 是 “全国核心”（全国 / 国际互联）|
|PA/PB/PC|均属于省 / 城域层级：PA（城域汇聚）→ PB（省骨干汇聚）→ PC（省骨干核心），是 “汇聚→核心” 的递进关系|

## 四、从局域网访问互联网的完整转发路径（以家庭宽带为例）

以成都某小区用户（局域网）通过 CMnet 访问百度（互联网）为例，流量走的路径：`家庭电脑（终端）` → 家庭交换机 → `AR/光猫（局域网出口）` → 小区 OLT/ONU（光纤接入） → `BRAS（接入网认证限速）` → `PA（城域汇聚）` → `BR（城域边界）` → `PB（省骨干汇聚）` → `PC/CR（四川骨干核心）` → `NCR（全国核心，如北京）` → 网间 BR（与电信互联，百度服务器在电信机房） → 百度服务器

若访问移动云资源（移动云专网），路径简化：`局域网` → AR → BRAS → `MB（移动云接入节点）` → 移动云数据中心（如成都移动云节点）

## 五、总结：设备 - 层级 - 功能对应表（核心速查）

| 设备    | 所属层级      | 核心功能                 |
| ----- | --------- | -------------------- |
| AR    | 局域网出口     | 企业 / 园区路由、DHCP、接入运营商 |
| BRAS  | 接入网       | 用户认证、带宽控制、流量汇聚       |
| MA    | 城域网汇聚     | 城域接入流量收敛             |
| MB    | 城域网核心     | 移动回传、移动云专网接入         |
| BR    | 城域 / 网间边界 | 路由互联、流量隔离            |
| PB    | 省骨干汇聚     | 全省城域流量收敛             |
| PC/CR | 省骨干核心     | 省内 / 省际流量转发          |
| NCR   | 全国骨干核心    | 全国 / 国际流量枢纽、跨运营商互联   |
| 网间 BR | 国内网间互联    | 与其他运营商对等互联           |

### 一句话核心逻辑：

**设备的名称（如 Aggregation 汇聚、Core 核心、Border 边界）已经暗示了其功能**，层级越靠用户侧（如 AR、BRAS），功能越偏向 “接入、认证、终端管理”；层级越靠核心（如 CR、NCR），功能越偏向 “高速转发、跨域互联”。掌握 “接入→汇聚→核心→互联” 的层级递进，就能快速理解所有设备的角色和关系。
