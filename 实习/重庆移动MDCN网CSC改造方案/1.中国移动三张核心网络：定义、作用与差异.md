城域网：Metropolitan Area Network 

## 一、中国移动三张核心网络：定义、作用与差异

### 1. 核心信息对比表

| 网络名称       | 英文全称（常用）                        | 中文定义                           | 核心作用                                                 | 承载业务                                                                                                 | 关键特性                                                                                        |
| ---------- | ------------------------------- | ------------------------------ | ---------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **云专网**    | Cloud Private Network           | 面向政企客户的 “专线 + 云” 一体化专用网络       | 为政府、企业提供端到端的专用通信通道，连接客户内网与中国移动云资源（如移动云 CMEC）或跨地域分支机构 | - 政企专线（办公互联、生产数据传输）<br><br>- 云资源访问（私有云、公有云专线接入）<br><br>- 行业解决方案（政务、金融、能源等涉密 / 高可靠业务）                 | - 专用通道，物理 / MPLS VPN 隔离，安全性极高<br><br>- 按需定制带宽（10M~100G）、时延保障<br><br>- 支持 “网 + 云 + 应用” 一站式服务 |
| **IP 承载网** | IP Bearer Network（含 IP RAN/SPN） | 中国移动内部核心支撑网络，承载各类 “关键业务” 的传输通道 | 作为运营商的 “骨干血管”，保障移动网、固网、运维系统等核心业务的稳定传输                | - 移动网（5G/4G 基站信号回传）<br><br>- 固网宽带（家庭 / 企业宽带的骨干传输）<br><br>- 内部支撑系统（BOSS、OA、NMS 等）<br><br>- 政企专线的骨干段传输 | - 高可靠性（双归属、自愈保护，99.999% 可用性）<br><br>- 低时延（毫秒级）、低丢包率<br><br>- 采用 MPLS VPN 隔离不同业务，QoS 优先级管控   |
| **CMNET**  | China Mobile Network            | 中国移动公众互联网骨干网                   | 为个人、家庭、企业用户提供互联网访问服务，连接国内 / 国际互联网                    | - 家庭宽带上网（网页浏览、视频、游戏）<br><br>- 移动数据上网（手机 4G/5G 访问互联网）<br><br>- 企业公网访问（非涉密的互联网业务）                      | - 面向公众，带宽资源共享<br><br>- 覆盖全国，连接国际出口（如中美、中欧海底光缆）<br><br>- 按公众互联网 QoS 策略调度，时延 / 稳定性低于专用网络      |

### 2. 三张网络的核心区别

- **服务对象**：云专网→政企客户（专用）；IP 承载网→运营商内部 + 核心业务（自用）；CMNET→公众用户（共享）
- **隔离性**：云专网（物理 / VPN 隔离）> IP 承载网（MPLS VPN 隔离）> CMNET（无隔离，共享带宽）
- **可靠性**：IP 承载网（核心支撑）≥ 云专网（政企承诺 SLA）> CMNET（公众尽力而为）

## 二、运营商网络分层架构（含 AR/CR/BR/PA/PC 定义）

运营商网络按「**地理范围**」分为：城域网（单城市）、省骨干网（全省）、全国骨干网（跨省份）；按「**功能层次**」分为：接入层、汇聚层、核心层（数通经典三层架构）。以下结合设备缩写和实际部署逻辑解析：

### 1. 分层架构总览表

| 地理范围       | 功能层次 | 核心设备（缩写 + 全称）                                                  | 设备作用                         | 典型部署场景                                                              |
| ---------- | ---- | -------------------------------------------------------------- | ---------------------------- | ------------------------------------------------------------------- |
| 城域网（单城市）   | 接入层  | AR（Access Router，接入路由器）<br><br>PA（Primary Access，主接入节点）        | 直接连接用户终端 / 客户内网，完成接入认证、带宽限制  | - 家庭宽带（PON+AR）<br><br>- 企业内网出口（政企专线接入 AR）<br><br>- 基站接入（5G 基站连接 PA） |
|            | 汇聚层  | BR（Backbone Router，汇聚路由器）<br><br>PC（Primary Convergence，主汇聚节点） | 汇聚接入层设备流量，进行路由转发、QoS 调度、故障隔离 | - 多个 AR/PA 连接到一个 BR/PC<br><br>- 城域内流量本地转发（如同一城市的企业互访）               |
|            | 核心层  | CR（City Core Router，城域核心路由器）                                   | 城域网出口，连接省骨干网，转发跨城 / 跨省流量     | - 城域内核心流量汇聚<br><br>- 与省骨干网 CR 对接                                    |
| 省骨干网（全省）   | 核心层  | CR（Province Core Router，省骨干核心路由器）                              | 汇聚全省各城域网流量，连接全国骨干网           | - 省内跨城流量转发（如成都→绵阳）<br><br>- 与全国骨干网对接                                |
| 全国骨干网（跨省份） | 核心层  | NCR（National Core Router，全国核心路由器）                              | 连接各省骨干网，转发跨省份 / 国际流量         | - 跨省流量转发（如成都→北京）<br><br>- 连接国际出口路由器（如上海、广州国际出口）                     |

### 2. 关键设备缩写详解（数通实践高频考点）

- **AR（接入路由器）**：运营商网络的 “最后一公里接入网关”，直接面对用户，支持 PPPoE 认证（家庭宽带）、VPN 接入（企业专线）、ACL 访问控制，是私网 / 用户终端接入运营商网络的第一个核心设备。
- **BR（汇聚路由器）**：接入层的 “流量汇总器”，一个 BR 可连接多个 AR，减少核心层设备的连接压力，同时实现流量隔离（如不同区域的 AR 流量互不影响），故障时仅影响局部接入用户。
- **CR（核心路由器）**：分城域 CR 和省 CR，是网络的 “交通枢纽”，转发能力极强（T 级带宽），负责大规模流量的高速转发，无本地接入功能，仅做路由交换。
- **PA/PC**：中国移动城域网中常用的接入 / 汇聚节点缩写，PA 侧重 “接入侧”（类似 AR 的扩展节点），PC 侧重 “汇聚侧”（类似 BR 的功能），通常部署在城市的区县或重要园区。

### 3. 分层架构的核心目的

- 隔离风险：接入层故障（如某个 AR 断网）仅影响局部用户，不扩散到核心层；
- 便于运维：按层次部署设备，配置、排障更高效（如汇聚层负责本地流量，核心层负责跨域流量）；
- 提升扩展性：新增用户 / 业务时，只需扩容接入层 / 汇聚层，无需改动核心层架构。

## 三、私网出网后的路由路径（从终端到 CMNET / 云专网）

“私网” 指企业内网、家庭局域网、实验室网络等，出网路径核心逻辑：**私网→接入层→汇聚层→核心层→目标网络**，具体分场景详解（结合中国移动实际部署）：

### 1. 场景 1：家庭用户（私网）访问互联网（CMNET）

- 关键说明：家庭用户通过 PON 技术接入 AR，AR 完成 PPPoE 认证后，流量经汇聚层、城域核心层进入 CMNET，最终访问互联网。

### 2. 场景 2：企业用户（私网）通过云专网访问移动云

- 关键说明：企业通过专线（如光纤）连接 AR，AR 将流量封装到 MPLS VPN 隧道（云专网专用通道），经汇聚层、核心层直达移动云，全程隔离公网，安全性高。

### 3. 场景 3：企业用户（私网）同时访问互联网（CMNET）和内网（跨地域分支）

- 关键说明：AR 支持双链路转发，通过路由策略区分流量（内网互访走云专网，公网访问走 CMNET）。

### 4. 核心路径总结

无论哪种场景，私网出网的**通用路径**是：`私网终端 → 接入层（AR/PA） → 汇聚层（BR/PC） → 城域核心层（CR） → 省骨干层（CR） → 目标网络（CMNET/云专网/IP承载网）`

- 跨省份访问时：需经过全国骨干网（NCR）转发（如成都企业访问北京服务器：成都省骨干→全国骨干→北京省骨干→目标网络）；
- 国际访问时：CMNET 流量经全国骨干网进入国际出口（如上海国际出口），连接国际互联网。

## 四、广域网与城域网的核心区别（补充认知）

|对比维度|城域网（MAN）|广域网（WAN）|
|---|---|---|
|覆盖范围|单个城市（如成都市区 + 郊区）|跨城市 / 省份 / 国家（如四川→北京、中国→美国）|
|核心作用|本地流量接入与转发|跨地域流量传输|
|包含层次|接入层、汇聚层、城域核心层|省骨干层、全国骨干层、国际出口层|
|典型设备|AR、BR、城域 CR、PA、PC|省 CR、NCR、国际出口路由器|
|对应运营商网络|中国移动成都城域网|中国移动省骨干网、全国骨干网|
