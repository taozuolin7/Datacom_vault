# 一级 RR 与二级 RR 的核心区别（表格对比）

| 对比维度                  | 一级 RR（Level-1 RR）                                           | 二级 RR（Level-2 RR）                                       |
| --------------------- | ----------------------------------------------------------- | ------------------------------------------------------- |
| **层级定位**              | 顶层路由中枢（跨区域 / 全网级）                                           | 区域级路由汇聚点（本地 / 省 / 片区级）                                  |
| **部署位置**              | 核心层（如运营商集团总部、全国骨干节点）                                        | 汇聚层（如省分公司、区域核心节点）                                       |
| **连接对象**              | 仅与二级 RR 建立 IBGP 邻居（作为二级 RR 的反射器）；一级 RR 之间通常全互联（或组成 Cluster） | ① 区域内的 BGP 路由器（作为客户端）；② 一级 RR（作为自己的反射器）                 |
| **路由流向**              | 接收各二级 RR 反射的 “区域汇总路由”，再反射给其他一级 RR 或跨区域二级 RR                 | 接收区域内客户端的 “明细路由”，汇总后反射给一级 RR；同时接收一级 RR 的跨区域路由，反射给本区域客户端 |
| **核心作用**              | 跨区域路由传递、全网路由统筹，隔离区域内细节                                      | 区域内路由汇聚、减少一级 RR 的路由条目压力，简化本地拓扑                          |
| **路由条目类型**            | 以 “区域汇总路由” 为主（减少路由数量）                                       | 以 “区域内明细路由” 为主，向上传递时会汇总                                 |
| **Cluster ID（集群 ID）** | 同一层级的一级 RR 可配置相同 Cluster ID（避免环路）                           | 每个二级 RR（或区域）配置独立 Cluster ID（与其他区域区分）                    |
| **适用场景**              | 全国 / 全网级网络（如运营商 AS、跨国家企业网）                                  | 省 / 市 / 片区级网络（如运营商省分公司、区域分公司网络）                         |
